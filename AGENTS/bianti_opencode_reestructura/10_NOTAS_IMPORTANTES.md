# Notas importantes para Jesús

## Por dónde empezar

Empezá por el Prompt 1: Auditoría.

No arranques tocando diseño de una. Primero OpenCode tiene que leer el proyecto y decir qué encontró.

Después seguí con el Prompt 2: Sidebar y layout.

Recién después armás cada pantalla.

## Cómo trabajar con OpenCode

No le pegues todo el prompt maestro y esperes magia.

Usalo así:

1. Pegás el prompt de auditoría.
2. Leés qué encontró.
3. Pegás el prompt de sidebar.
4. Probás.
5. Seguís con Panel principal.
6. Probás.
7. Seguís con Categorías.
8. Probás.
9. Seguís con Pedidos.
10. Probás.
11. Seguís con Contabilidad.
12. Probás.
13. Seguís con Métricas.
14. Probás.
15. Seguís con Estadísticas.
16. Recién ahí pedís prueba general.

## Regla de oro

Una pantalla por vez.

Si OpenCode modifica 20 archivos juntos sin control, después encontrar el error es como buscar una media negra en un placard apagado.
