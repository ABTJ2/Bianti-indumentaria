# SKILL - PROTECCIÓN DE FUNCIONALIDAD

## Objetivo
Evitar que los cambios visuales rompan el sistema.

## Reglas
- Antes de cambiar un ID, buscarlo en todo el proyecto.
- Antes de eliminar una clase, revisar si JS la usa.
- Antes de mover un script, revisar dependencias.
- No tocar Supabase si no es necesario.
- No cambiar tablas ni columnas.
- Probar después de cada cambio importante.
