# Requerimientos funcionales BIANTI Admin

## RF-01 Login admin
El sistema debe permitir el ingreso del administrador usando la tabla `usuarios`.

## RF-02 Sidebar único
El sistema debe mostrar el mismo sidebar en todas las pantallas admin.

## RF-03 Panel principal
El sistema debe mostrar un resumen general del flujo de catálogo, consultas y ventas.

## RF-04 Categorías
El sistema debe permitir gestionar categorías del catálogo.

## RF-05 Pedidos como consultas WhatsApp
El sistema debe guardar o mostrar consultas de WhatsApp como pedidos en revisión.

## RF-06 Decisión del admin
El administrador debe poder marcar una consulta como:
- vendido
- no_vendido

## RF-07 Impacto contable
Solo los pedidos vendidos deben impactar en contabilidad.

## RF-08 Métricas
El sistema debe mostrar rendimiento de consultas y ventas.

## RF-09 Estadísticas
El sistema debe mostrar análisis de productos vendidos, consultados y destacados.

## RF-10 Catálogo público
El catálogo público debe seguir funcionando y mostrando productos visibles/disponibles.

## RF-11 Productos
La gestión de productos no se elimina. Puede quedar como acceso rápido, modal o pantalla secundaria, pero no como ítem principal del sidebar.

## RF-12 Supabase
El sistema debe usar las tablas actuales de Supabase.
